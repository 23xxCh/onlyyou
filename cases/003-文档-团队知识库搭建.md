# 案例：用飞书知识库搭建团队 Wiki

## 场景
用户团队的文档散落在各处，想建一个统一的知识库。

## 解决方案

### 1. 创建知识空间
- 按团队/部门创建知识空间
- 设置空间管理员和成员权限

### 2. 规划目录结构
```
📚 团队知识库
├── 🏠 新人指南
│   ├── 公司介绍
│   ├── 入职流程
│   └── 常用工具
├── 📋 工作流程
│   ├── 需求流程
│   ├── 发布流程
│   └── 审批流程
├── 📖 技术文档
│   ├── 架构设计
│   ├── API 文档
│   └── 运维手册
└── 📝 会议纪要
    ├── 周会
    └── 评审会
```

### 3. 权限管理
- 空间级别：管理员/编辑者/阅读者
- 节点级别：可以单独设置
- 支持分享给外部人员

### 4. 文档模板
- 创建常用模板（会议纪要、需求文档、周报）
- 新建文档时直接选模板
- 保持格式统一

## 最佳实践
- 知识库不是越多越好，关键是结构清晰
- 定期清理过期文档
- 用"目录"节点组织结构，不要全平铺
- 文档标题要规范，方便搜索
