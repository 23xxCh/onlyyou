# 案例：用飞书日历做高效会议管理

## 场景
用户团队每天有很多会议，经常冲突，也不知道怎么查谁有空。

## 解决方案

### 1. 创建日程时加参会人
- 创建日程时添加所有参会人
- 飞书会自动检查每个人的忙闲状态
- 绿色=空闲，灰色=忙碌

### 2. 使用会议室资源
- 在日程中添加会议室
- 避免会议室冲突
- 支持查看会议室使用情况

### 3. 设置会议提醒
- 默认提醒：15 分钟前
- 重要会议：1 小时前
- 支持自定义提醒时间

### 4. 重复日程
- 周会、日站会设为重复日程
- 用 RRULE 格式支持复杂重复规则
- 修改单次 vs 修改全部

### 5. 忙闲查询 API
- 批量查询 1-10 人的忙闲状态
- 适合自动排会议的场景
- 时间范围限制：需要合理设置

## 最佳实践
- 会议必须有明确的议程（写在描述里）
- 30 分钟会议 > 1 小时会议
- 用飞书会议的"纪要"功能自动生成会议记录
- 会后用任务功能分配 action item
