# 案例：用多维表格从零搭建项目管理

## 场景
用户是一个创业团队的 PM，想用飞书管理项目进度，但不知道从哪开始。

## 解决方案

### 1. 创建多维表格
- 创建一个 Bitable，命名为"项目管理看板"
- 创建以下字段：

| 字段名 | 类型 | 说明 |
|---|---|---|
| 任务名称 | 文本 | 任务标题 |
| 负责人 | 人员 | 指派的成员 |
| 状态 | 单选 | 未开始/进行中/已完成/已延期 |
| 优先级 | 单选 | P0/P1/P2/P3 |
| 截止日期 | 日期 | 任务截止时间 |
| 所属项目 | 单选 | 项目名称 |
| 备注 | 文本 | 补充说明 |
| 创建时间 | 创建时间 | 自动记录 |
| 完成时间 | 修改时间 | 自动记录 |

### 2. 创建视图
- **看板视图**：按状态分组，拖拽管理任务流转
- **甘特图视图**：看项目时间线
- **表格视图**：默认视图，适合批量编辑

### 3. 设置自动化
- 当状态改为"已完成"时，自动填写完成时间
- 当截止日期临近时，自动提醒负责人

## 最佳实践
- P0 任务不超过 3 个，保持聚焦
- 每周回顾看板，清理过期任务
- 用筛选视图让每个人只看自己的任务
