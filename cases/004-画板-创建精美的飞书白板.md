# 案例：用飞书画板创建精美的可视化内容

## 场景
用户想把一个复杂的技术架构、流程说明或会议总结变成可视化的飞书画板，嵌入文档中。

## 解决方案

### 前置条件
1. Node.js 20+
2. `npm install -g @larksuite/cli`
3. `lark-cli config init`（扫码登录）
4. `lark-cli auth login`（授权飞书账号）
5. `@larksuite/whiteboard-cli`（通过 npx 自动下载，无需手动安装）

### 工作流程

1. **理解内容**：用户想要什么内容的画板？用途是什么？受众是谁？
2. **选择风格**：根据场景推荐配色方案
   - 正式/严肃 → Restrained 级别（如 Monochrome、Grove、Jade Lens）
   - 通用/现代 → Balanced 级别（如 Coral、Pin & Paper、Riptide Cobalt）
   - 活泼/有趣 → Bold 级别（如 BlockFrame、Riso Brut、Crayon Stack）
3. **生成 SVG**：用原生形状（rect、circle、text）构建，遵守飞书画板渲染规则
4. **渲染验证**：检查文字溢出、边距、重叠等问题
5. **写入飞书**：将画板写入飞书文档，返回文档链接和渲染图片

### 飞书画板渲染规则（关键点）

- **只能用原生形状**：rect、rounded rect、circle、ellipse、直线、polyline、text
- **不能用**：渐变、滤镜、透明度、贝塞尔曲线路径
- **箭头**：用 marker-end 属性，不能手动画箭头
- **字体**：固定 Noto Sans SC，不要设置 font-family
- **颜色**：不支持透明度，用纯色浅色代替
- **阴影**：不支持模糊阴影，用偏移实色复制实现
- **画布**：逻辑坐标 1600-1700 宽，无固定比例

### 35 种配色方案速查

| 级别 | 风格数 | 代表风格 |
|---|---|---|
| Restrained（克制） | 9 | Monochrome、Grove、Jade Lens |
| Balanced（平衡） | 15 | Coral、Pin & Paper、Riptide Cobalt |
| Bold（大胆） | 11 | Riso Brut、BlockFrame、Crayon Stack |

## 使用示例

```
用户："帮我把系统架构画成飞书画板，用 Riso Brut 风格"
龙虾：理解架构 → 选择 Riso Brut 配色 → 用原生形状构建 SVG → 渲染验证 → 写入飞书文档 → 返回链接+图片
```

## 最佳实践

- 内容优先，不要在画板上放元数据（来源、风格名等）
- 文字大小 ≥ 16px，确保可读性
- 每次渲染后检查图片，发现布局问题就地修复
- 告诉用户可以随时切换风格，用同样的内容换配色
